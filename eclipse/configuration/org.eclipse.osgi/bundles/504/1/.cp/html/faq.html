IAR Embedded Workbench for Eclipse FAQ

Network problems

Eclipse says "unable to connect to repository". Can I install any other way?

Yes. All the update sites are available as downloadable zip files. Open http://eclipse-update.iar.com/ and select the "zip file" link besides your product.

Dependency problems

IAR Plugin installation fails with missing component error

If Eclipse fails installation with an error message similar to:

Cannot complete the install because one or more required items could not be found. Software being installed: IAR Project Importer (EXPERIMENTAL) 0.11.3.v201111110257 (com.iar.common.projectimport.feature.feature.group 0.11.3.v201111110257) Missing requirement: IAR Project Importer Components 1.0.0.v201111110257 (com.iar.common.projectimport 1.0.0.v201111110257) requires 'bundle org.eclipse.emf.ecore 2.6.1' but it could not be found

this is most probably due to an error with the Eclipse mirror which should provide the org.eclipse.emf.ecore 2.6.1 bundle.

As this problem is caused by a misconfiguration in the Eclipse.org servers (see this bugzilla entry), the easiest way to solve this issue is to simply wait and retry the installation at a later time.

Otherwise, it is possible to force Eclipse to download updates from a specific mirror by doing the following (for Eclipse 3.7 Indigo release):

  • Disabling the http://download.eclipse.org/releases/indigo update site in Window > Preferences > Install/Update > Available Software Sites
  • Adding the following mirror URL to the list of available update sites in the same preference page: http://ftp-stud.fht-esslingen.de/pub/Mirrors/eclipse/releases/indigo/

For the older Eclipse Helios 3.6 distributions, the URLs to disable / add are, respectively:

  • http://download.eclipse.org/releases/helios
  • http://ftp-stud.fht-esslingen.de/pub/Mirrors/eclipse/releases/helios/
